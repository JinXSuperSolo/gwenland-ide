# AI Chat Pane Polishing (M8): borderless pane, custom controls, thinking display, and human-gated diff review

- **Date:** 2026-06-22
- **Issue:** GWEN-314, GWEN-315, GWEN-316, GWEN-317, GWEN-318 (M8 ŌĆö AI Chat Pane Polishing)
- **Milestone:** Milestone 8 ŌĆö AI Chat Pane Polishing

## Problem / Context

The Milestone 4 AI pane worked but looked like a generic form panel bolted onto the IDE: hard borders between every section, cramped padding, native `<select>` dropdowns for provider/model/conversation, a boxy "Send" button, and terse empty states (`Open a folder to use AI chat.`). It also had no way to show a model's reasoning, and AI code suggestions arrived as plain fenced blocks with no structured way to apply them.

M8 turns that pane into a cleaner, review-oriented assistant surface around one interaction model: **AI suggests, the user decides.** Nothing is auto-applied ŌĆö accepted hunks are the only changes written to disk. The work kept every M4 architectural boundary intact (pure/provider-neutral logic in `engine/`, Tauri command + app state in `frontend/src/main.rs`, Svelte state/UI in `frontend/ui/src/lib/`, editor extensions in `frontend/ui/src/lib/editor/`), stayed **dark-theme only**, reused the existing `Icon.svelte` registry rather than adding a second icon system, kept secrets in the OS keychain (never frontend state), and added **no new Rust dependencies**.

It was taken as six dependency-ordered waves, each ending with `pnpm check` + `pnpm build` (and `cargo test` where the engine changed) green and prior behaviour non-regressed: (1) borderless layout + M8 tokens, (2) empty states, (3) custom dropdowns + reasoning selector, (4) thinking parsing + display, (5) DiffParser + detection + system prompt, (6) review mode + CodeMirror overlay + hunk actions + panels.

## Change

### Wave 1 ŌĆö Borderless layout and M8 visual tokens

The 13 final M8 semantic values (backgrounds, primary/primary-light, muted text, subtle border, thinking tint, added/removed diff colors) were added as **AI-scoped `--ai-*` variables on `.ai-panel`** rather than mutating any global token in `tokens.css` ŌĆö they inherit to every pane child (messages, conversation, future thinking/review components). `AiPanel.svelte` lost its `border-bottom`/`border-top` section separators; the header is now 40px with the `AI` title left and compact right-side icon actions (new conversation, settings, close); the message list gained breathing room (12/16px padding) and a muted 4px scrollbar; the composer became a rounded floating surface with a circular arrow send control (new `arrow-up` icon) that keeps the existing stop/cancel and attachment-chip behaviour. `AiConversation.svelte` and `AiMessage.svelte` got the same borderless, token-aligned treatment.

### Wave 2 ŌĆö Empty state variants

The single terse empty state became two warm variants in `AiPanel.svelte`: a no-workspace state (sparkle icon, `Open a folder to start`, subtext, and an `Open Folder` CTA wired to the existing `openFolder()` dialog wrapper ŌĆö subtle orange outline) and a workspace-open state (chat icon, `Ask anything about your code`, plus three suggestion chips: `Explain this file`, `Find bugs`, `Write a test`). Clicking a chip ensures a conversation exists (creating one so the composer is enabled), pre-fills the composer, and focuses it. All three legacy strings (`Open a folder to use AI chat.`, `Create a conversation first`, `Create a conversation with ŌĆ” above to begin.`) were removed.

### Wave 3 ŌĆö Custom dropdowns + reasoning selector

A reusable `Dropdown.svelte` primitive replaced native `<select>`s: a pill trigger + dark floating card using the single-choice `menu`/`menuitemradio` pattern, with `--ai-primary-light` + check indicator on the selected option, a subtle fade/scale (`gw-anim-pop`), outside-click + Escape close, and full keyboard nav (Ōåæ/Ōåō move, Enter select, Tab close). It backs `ProviderDropdown`/`ModelDropdown` (the model picker keeps a styled manual-id input fallback when listing is unavailable) and the conversation picker ŌĆö so **no native select remains in the AI pane**. A `ReasoningLevelSelector` (Low/Medium/High/Extra High) appears only for thinking-capable models, gated by a documented heuristic (`reasoning.ts`): Anthropic `claude-3-7*` and `claude-{opus,sonnet,haiku}-4+`, plus any provider whose model id contains `deepseek-r1`/`qwen3`/`qwq`. The level is stored in AI chat state; provider/model switching still flows through the existing `setProvider`/`setModel`/settings-persistence paths, and the dropdowns store only the existing `hasKey` boolean ŌĆö no keys, tokens, or headers.

**Refinement (post-Wave 3, on UI feedback):** the three separate top-row pills were merged into a single composer-integrated **`ComposerModelMenu`** ŌĆö one `Model ┬Ę Reasoning` pill that opens upward into one sectioned popup (Reasoning / Provider / Model), matching a Codex-style chat input. The three wrapper components were deleted; the generic `Dropdown` stays for the conversation picker.

### Wave 4 ŌĆö Thinking parsing and the thinking block

A chunk-safe `<think>ŌĆ”</think>` parser (`thinking-parser.ts`) splits reasoning from the answer, holding partial tags across stream-chunk boundaries so raw tags never render and thinking is never duplicated into the final answer. It runs both while streaming and when loading persisted turns (`splitThinking`), so old JSONL stays compatible. `ChatMessage` gained a runtime `ThinkingState` (content, streaming, start/end timestamps, collapsed); thinking auto-collapses on the first answer token. `ThinkingBlock.svelte` renders before the answer ŌĆö `ThinkingŌĆ”` while streaming (200px cap + auto-scroll), then auto-collapsed `Thought for <duration>`, with a 2px primary left accent, thinking-tint background, and Inter (not mono) typography. For structured providers, the **engine adapters wrap reasoning into the same `<think>` text stream** (Anthropic `thinking_delta`; OpenAI/DeepSeek `reasoning_content`; Ollama `thinking`) ŌĆö so one frontend parser handles every provider with **no new Tauri event fields and no new Rust deps**, degrading cleanly when no reasoning is present.

### Wave 5 ŌĆö DiffParser, detection, and system prompt

A pure `engine/src/ai/diff.rs` parses unified diff text into serializable `DiffFile`/`DiffHunk`/`DiffLine` DTOs (re-exported from `ai/mod.rs`, zero Tauri imports, only `serde`). It tolerates prose, ```` ```diff ```` fences, multi-file diffs, added/removed-only hunks, and empty lines; count-driven hunk bodies disambiguate removed lines that start with `-`; malformed hunk headers return a safe error instead of panicking. A GwenLand system prompt was added in `main.rs` and passed via `MessageRequest.system` ŌĆö it includes the exact unified-diff format block and the local-first / user-decides / no-auto-apply guidance. A `parse_diff` Tauri command plus `diff-detection.ts` gate the parser behind a cheap marker check and only treat a response as a proposal when Ōēź1 file with Ōēź1 hunk parses; failures are non-destructive (raw text stays, a muted notice shows).

### Wave 6 ŌĆö Review mode, CodeMirror overlay, hunk actions, panels

A `diff-review.ts` store holds the proposal as files ŌåÆ hunks with per-hunk status (`pending`/`accepted`/`rejected`/`failed`), the active file/hunk, and summary counts ŌĆö runtime only, never persisted to JSONL. A CodeMirror overlay (`diff-overlay.ts`) renders the diff for the active file **without mutating the document**: a **separate `StateField`** (so it never clears M6 lint diagnostics) provides removed-line backgrounds + gutter classes and per-hunk **block widgets** showing the added lines plus inline Accept/Reject controls. Hunks are located by **content search** (not raw line numbers), so the overlay and apply survive unrelated edits. Accept/Reject *stage* a decision; when all hunks resolve (or via Accept All), accepted hunks apply in one validated pass per file and only changed files are saved ŌĆö the active file via the live editor view (preserving undo/user edits), open files refreshed, unopened files written directly; unmatched hunks are marked failed without corrupting the file; Cancel/Escape exits with no writes. A floating `DiffActionBar` over the editor shows `Review Changes` + counts + Accept All/Reject All/Cancel and hosts the `]`/`[`/`a`/`r`/`Escape` shortcuts (blocked in the AI composer/model inputs, allowed in the editor where review happens). A `DiffReviewPanel` in the AI pane lists changed files with per-file progress and jumps the editor to a file on click. Exiting clears the store, which clears the overlay and hides the bar/panel ŌĆö no ghost decorations.

### Test isolation fix (engine)

The `ai::conversation` tests share the process-global `GWENLAND_HOME` env var and assumed single-threaded execution; under the default parallel `cargo test` they clobbered each other. They now serialize via a process-wide `TEST_LOCK` held for each `TestEnv`'s lifetime (poison-tolerant), so the full suite passes in parallel.

## Why this approach

Scoping the M8 colors to `.ai-panel` rather than redefining global tokens kept the rest of the IDE's theme untouched while still letting every pane child inherit them ŌĆö and avoided a risky global find-replace. The biggest design lever was the **unified `<think>` text stream**: instead of inventing a second event channel and DTO for structured reasoning, the engine wraps Anthropic/Ollama/DeepSeek reasoning in `<think>` tags inside the existing token stream, so a single chunk-safe frontend parser covers every provider, the Tauri event contract is unchanged, and persistence/loading reuse the same split ŌĆö all with zero new Rust deps. Parsing diffs in the engine as pure DTOs keeps one reliable representation near the provider code and testable in isolation. For review, keeping the overlay in a **separate CodeMirror StateField** is what guarantees M6 diagnostics survive, and **content-based hunk location** (vs. trusting line numbers) plus a **staged-then-batched apply** is what makes accept/reject safe: nothing touches a file until the user commits, a hunk that no longer matches fails loudly instead of corrupting, and Cancel is genuinely write-free.
